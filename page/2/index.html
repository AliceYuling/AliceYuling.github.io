Service Worker学习小结

什么是 Service Worker

Service Worker 是一段运行在浏览器后台中的脚本，它独立于当前页面，提供了那些不需要与web页面交互的功能在网页背后悄悄执行的能力。 [1] Service Worker 是 PWA 当中的一部分，它是一个可编程的网络代理，可以控制 Web 页面的所有网络请求。 Service Worker 独立于 JavaScript 主进程，它无法直接访问 DOM 和 window 对象。Service Worker具有以下作用：

  • 浏览器缓存
  • 消息推送
  • 实现离线 Web App

Service Worker 具有独立的生命周期， 如下图所示：

Service Worker 具有下载、安装、激活三个生命周期。

  • 下载： 用户首次访问Service Worker控制的网站时，Serivce Worker会被下载

  • 安装：下载完成后，开始安装阶段，通常在安装阶段进行一些资源的加载和缓存

  • 激活： 安装完成后，开始激活 Service Worker，这个阶段可以对之前缓存的资源的管理。激活之后，Service Worker 即开始对其所属范围内的页面进行控制。

docker基础学习

Docker

Dockers是是一个开源的应用容器引擎，让开发者将应用以及依赖包打包到一个可移植的容器中，然后发布到任何流行的 Linux或Windows 机器上，也可以实现虚拟化。

docker 镜像

Docker 镜像是一个特殊的文件系统，除了提供容器运行时所需的程序、库、资源、配置等文件外，还包含了一些为运行时准备的一些配置参数（如匿名卷、环境变量、用户等）。镜像不包含任何动态数据，其内容在构建之后也不会被改变。

docker 容器

容器就是将软件打包成标准化单元，以用于开发、交付和部署，是镜像运行的实体。
容器和虚机的区别：传统虚拟机技术是虚拟出一套硬件后，在其上运行一个完整操作系统，在该系统上再运行所需应用进程；而容器内的应用进程直接运行于宿主的内核，容器内没有自己的内核，而且也没有进行硬件虚拟。因此容器要比传统虚拟机更为轻便。

仓库

Docker Registry 是一个集中的存储、分发镜像的服务。通常，一个仓库会包含同一个软件不同版本的镜像，而标签就常用于对应该软件的各个版本。我们可以通过 <仓库名>:<标签> 的格式来指定具体是这个软件哪个版本的镜像。如果不给出标签，将以 latest 作为默认标签。

Docker Registry 公开服务是开放给用户使用、允许用户管理镜像的 Registry 服务。一般这类公开服务允许用户免费上传、下载公开的镜像，并可能提供收费服务供用户管理私有镜像。最常使用的 Registry 公开服务是官方的 Docker Hub，这也是默认的 Registry，并拥有大量的高质量的官方镜像。除此以外，还有 CoreOS 的 Quay.io，CoreOS 相关的镜像存储在这里；Google 的 Google Container Registry，Kubernetes 的镜像使用的就是这个服务。

grid布局

Grid（网格）布局，顾名思义，它把页面划分成一个个网格，可以自由控制网格大小、位置以达到不同的布局样式。
网格是由一组水平垂直相交的直线所形成的，水平网格线划分出行，垂直网格线划分出列。

1. 容器属性

1.1 display

display: grid; 指定一个元素采用网格布局。在一个元素上应用了 display: grid; 则这个元素为容器元素，其内部的子元素为网格元素。也可以设定为 display: inline-grid; 则容器元素为行内元素。

1.1 grid-template

grid-template 属性包含 grid-template-rows, grid-template-columns 和 grid-template-areas。

grid-template-rows, grid-template-columns 分别用于设置各行和各列的行高、列宽，其值可以是 px、百分比或者 fr。 fr 是网格布局中的一个布局单位，用于在一系列长度空间中分配剩余空间, 如指定 grid-template-columns: 100px 2rf 1fr;, 那么在除去第一列的剩余空间中分配第二列与第三列的宽度，第二列的宽度是第三列的两倍。

节流和防抖函数

前两天写轮播图的时候，发现了一个问题就是在快速多次点击翻页按钮的时候，会多次触发翻页函数，此时由于上一次翻页的动画还没结束，就会出现很多问题。这个时候就需要用到节流函数。节流和防抖这两个概念经常都是一起出现的，自己对这两个概念和方法也不是很熟，在这篇博客里就放到一起写了。

节流/防抖的概念及区别

节流(throttle)： 某个函数在一个特定的时间内只执行一次。比如调整窗口大小时，resize事件会持续被触发，使用节流函数可以让其在事件触发期间每间隔一段时间执行一次，而不是每触发一次事件执行一次事件函数。

防抖（debounce)： 在事件触发后一段时间t内没检测到事件时则开始执行事件函数，若在t内再次检测到了事件触发那么重新开始计时。也就是防抖是在一个动作结束后才执行。

下面这张图可以很形象的说明节流和防抖的区别，第一行是正常的事件触发频率，可以看到防抖是把在动作结束后一段时间内未再次检测到事件触发则执行，它可以把函数的多次调用压缩成一次。而节流则相当于减满了函数的执行速度。

图片来自 (http://demo.nimius.net/debounce_throttle/)

以下可能会高频触发的事件可能会导致页面卡顿、浏览器奔溃等情况出现，如果在事件函数中写了异步操作的话，可能会导致结果和预想的不一样，为了绕开这些问题，可以使用节流/防抖：

  • click
  • resize
  • input
  • scroll
  • mousemove
  • …

节流比防抖更适合用于一些触发更频繁的场景中，如动画相关的resize/scroll等，而防抖适合用于input/keyup

节流的实现

事件触发时，计算这次触发的时间与上次执行函数的时间之差，如果时间差达到设定的时间间隔，则执行这个函数，否则不执行。

防抖的实现

使用定时器，在每次事件触发时，清除定时器，直到设定的时间内没有事件触发，则函数被执行
